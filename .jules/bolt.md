## 2024-05-19 - O(N log N) Array Sorts for Maximum Values
**Learning:** The codebase previously contained an anti-pattern where an array was sorted entirely in an O(N log N) operation, solely to access the single largest/latest value using `[0]`. Given the frequent re-rendering nature of `app.js` and the potential for arrays like logs and metrics to become very large, this was an unnecessary CPU block.
**Action:** Always replace `.sort()[0]` or `.sort().reverse()[0]` implementations with a linear O(N) `.reduce()` loop when the only required outcome is extracting the maximum or minimum value in an array.
